# 思源 Inbox+

思源 Inbox+ 是一款强大的轻量级收集辅助插件，旨在将 Telegram 消息与思源笔记自带的「微信收集箱」内容无缝同步并沉淀为原生笔记块，支持灵活的长短内容分流路由，让碎片化信息收集更加高效。

## 🌟 核心特性

- **多源无缝同步**：支持 Telegram 消息与思源微信收集箱内容的双向或单向静默拉伸与同步。
- **智能分流路由**：
  - **短内容暂存**：将碎片化短内容自动追加写入指定的「短内容暂存文档」中。
  - **长内容分流**：当内容字数超过设定阈值（如 200 字）时，自动作为长文章移入指定笔记本，并独立建档。
- **灵活的 Telegram 路由**：可为 Telegram 消息配置专属目标文档，留空则自动沿用短内容文档。
- **原生块级写入**：所有同步结果直接使用思源原生块追加写入文档，无需额外面板，保留最纯粹的笔记体验。
- **静默后台轮询**：默认以 60 秒间隔自动轮询，支持随时改为自定义间隔、每日定时或手动刷新。

## ⚙️ 核心设置

- **同步控制**：默认「自动轮询 + 60 秒」，也可随时调整间隔，或改为每日定时、手动刷新。
- **短内容绑定文档**：碎片化短内容的追加目标文档。
- **长内容分流阈值**：判定长短内容的字数界限（默认 200 字）。
- **长内容目标笔记本**：长内容自动分流、独立建档的目标笔记本。
- **Telegram 绑定文档**：Telegram 专属目标文档；若留空则合并追加至短内容绑定文档。
- **Telegram Bot Token**：用于同步消息的 Telegram 机器人 Token。
- **授权用户名**：限制仅处理特定 Telegram 用户或 ID，确保收集隐私安全，留空则不限制。

## 🚀 简易使用

1. 在插件设置中配置好目标文档、笔记本以及 Telegram Bot Token 等信息。
2. 点击顶栏「Inbox+」图标即可立即手动触发同步，或者等待后台自动轮询。
3. 碎片信息会根据长短自动分流到相应的文档或笔记本中。

## 🛠️ 常见问题

### Telegram 同步偶发 `forward request failed` / `getUpdates` / `EOF`

这类报错通常表示插件向 Telegram Bot API 拉取消息时，网络连接在返回前被中断。请优先排查当前网络、系统代理、透明代理或网关到 `api.telegram.org` 的连通性。

如果此前已经可以正常通过 Telegram 同步笔记，且 Bot Token、授权用户名、代理等网络配置没有改动，这类错误多半是临时网络波动或连接复用中断，后续轮询可能会自动恢复。若持续出现，再检查代理规则、Telegram API 可达性和思源日志中的同一时间段网络错误。
